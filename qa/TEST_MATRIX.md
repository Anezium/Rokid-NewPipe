## Missing Hooks

- Keep the debug-only DUMP_ROKID_STATE command in sync as new Rokid-only overlays are added.
- Expand the uiautomator dump parser with explicit navigation steps for downloads, play queue, chapters, settings selectors, and notification action rows.
- Record real R08 key traces and add them to `RokidKeyMapperTest` if hardware emits key sequences beyond `RIGHT+DOWN`, `LEFT+UP`, `183`, `184`, and `202`.
- Confirm on hardware that Access Bridge calls the custom `Select` accessibility action while moving through player rail buttons and menu rows, then uses click only for tap activation.
- Confirm the standalone play queue activity uses the same one-axis R08 navigation as the main app and focuses queue rows instead of the RecyclerView container.
- Confirm RecyclerView navigation does not consume a forward swipe at the final row without moving focus; the next reachable control or wrap target must become focusable.
- Confirm tap/select activates the currently focused row/control even when Android focus lands on an inner text/icon child; it must not fall back to a stale previous target.
- Confirm tap/select activates long-click-only focus stops, such as context-menu rows, rather than leaving the R08 focus on an inert target.
- Confirm rows with secondary actions, such as move-down handles, expose both the primary row and secondary action in one-axis focus order.
- Confirm long-press/context action dialogs for video rows and play queue rows open as R08-navigable dialogs, not inaccessible popup menus.
- Confirm “add to playlist” list dialogs receive R08 key events and focus playlist rows/new-playlist controls.
- Confirm feed help/filter/error dialogs, subscription context dialogs, notification action dialogs, and history/settings confirmations receive R08 key events.
- Confirm ErrorActivity sharing opens the NewPipe R08 chooser rather than Android's system resolver.
- Confirm subscription import/export opens NewPipe R08 dialogs rather than Android overflow submenus.
- Confirm download item menus, download history confirmations, backup/restore prompts, external-player install prompts, and migration info dialogs receive R08 key events.
- Confirm Compose-hosted About/Licenses screens use one-axis R08 focus movement, debounce paired aliases, and map double tap to select.
- Confirm About/Licenses keep using `ComposeActivity`, not the removed legacy `.about.AboutActivity` manifest entry.
- Confirm AndroidX preference dialogs in Settings use NewPipe R08 dialogs, including list, multi-select, and text-entry preferences.
- Confirm app-language settings stay inside NewPipe in Rokid mode instead of launching Android system settings.
- Confirm Wi-Fi recovery opens Android Wi-Fi settings directly from the offline panel; other unavoidable Android settings launches, such as notifications, captions, sound, storage, and overlay permission, first show a NewPipe R08 confirmation dialog and return cleanly with Back after opening settings.
- Confirm unavoidable Android file/document pickers first show a NewPipe R08 confirmation dialog and return cleanly with Back after opening DocumentsUI.
- Confirm the standalone play queue speed action cycles through fixed speeds in Rokid mode instead of opening the slider-based playback-parameter dialog.
- Confirm on hardware that AlertDialog/FeedGroup windows receive R08 key events through their dialog key listener and that the keyboard overlay appears above those windows.
- Confirm the Rokid keyboard action key uses the current text-entry action label, not always "Search", across search, import URL, rename, create, and error-comment fields.
- Confirm the Rokid keyboard key labels and selected-key prefix are string-backed, not hardcoded English speech text.
- Confirm on hardware that DownloadDialog cycles media type, quality/format, audio track, and thread count with tap actions instead of opening spinner popups or touching the SeekBar.
- Confirm DownloadDialog announces each R08 control with string-backed control/value/action hints, not hardcoded English speech text.
- Confirm on hardware that FeedGroupReorderDialog can reorder using row tap move-down and persist with OK.
- Confirm on hardware that the app-owned FilePickerActivityHelper accepts the same one-axis R08 navigation as the rest of the app.
- Confirm on hardware that subscription import by channel URL opens the Rokid keyboard and the keyboard go action starts the import confirmation.
- Confirm on hardware that tabs and PeerTube instances can be reordered without drag by using their R08 move-down actions.
- Confirm on hardware that local playlists, bookmark playlists, and play queue rows can be reordered without drag by using their R08 move-down actions.
- Confirm on hardware that PlaybackParameterDialog, if reached in Rokid mode, uses dialog navigation and discrete +/- controls while its SeekBars are skipped by Accessibility.
- Confirm PlaybackParameterDialog uses string-backed action hints and selected/on/off states for Access Bridge speech.
- Confirm on hardware that snackbar action buttons receive focus when shown and can be activated by tap before focus returns to the previous surface.
- Confirm on hardware that share/open-with choices use the NewPipe R08 dialog chooser and launch the selected external app without losing URI grants.
- Confirm on hardware that browser/external-app links use the same NewPipe R08 dialog chooser when multiple handlers exist, and that the selected handler launches with selector state cleared while preserving URI grants.
- Confirm on hardware that reCAPTCHA WebView remains navigable enough to reach Done/Back; the challenge itself still depends on the provider's WebView accessibility.
- Add a network-on fixture/device step for player quality/speed/subtitles, because offline glasses cannot load streams.
- Add fixture dumps for secondary surfaces so `Assert-NoUnnamedFocusableActions` can be run without live hardware.
